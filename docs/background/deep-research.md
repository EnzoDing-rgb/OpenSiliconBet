## § 1 三家身世

**写作目标**：x86 / ARM / RISC-V 三个指令集各自怎么来的，把人物、关键年份、一句话定性讲清楚——为后续所有讨论铺垫"角色背景"。

**必答的关键问题**：

- x86 是哪一年、由谁、以什么产品起家？（涉及 Intel、AMD 历史关系）
- ARM 是哪一年、由哪几家合资成立？为什么取名 ARM？Apple 是不是创始股东之一？
- RISC-V 是哪一年、由哪个学校发起？背后的核心主张是什么（开源指令集）？

**建议字数**：400 字

<!-- PLACEHOLDER_START -->

**x86 — CISC 时代的奠基者，PC/服务器 50 年的统治者**：1978 年 Intel 发布 8086，开启了 CPU 指令集架构成体系的起点；80186/286/386/486 一路延续到 Pentium、酷睿、至强——每一代 CPU 都沿用从 8086 开始的指令集，保证老软件向后兼容，于是这些"八六系列"芯片被统称为 x86 架构，撑起了整个 PC 时代。它属于复杂指令集（CISC），优点是减少程序员写代码的复杂度、节省当时昂贵的内存空间；缺点是解码更难、高性能下的功耗与效率难以兼顾。AMD 后来通过授权拿到 x86-64 扩展资格，成为 Intel 的长期竞争者。耐人寻味的是，2024 年 Intel 和 AMD 这对老冤家**罕见地联手**，拉上 Microsoft 和 Google 成立"x86 生态咨询小组"（x86 Ecosystem Advisory Group），共同发布全新 x86S 架构标准——共同对抗 ARM 在数据中心的入侵。

**ARM — 1985 年从英国学院派走出，35 年只卖图纸不造芯片**：早在 1985 年英国 Acorn Computers 没有跟随 x86 的 PC 路线，而是基于 RISC 精简指令集设计了 ARM1 CPU——把复杂的单条指令拆解成多个精简指令、性能与功耗在当时非常亮眼。1990 年 Acorn 把芯片团队剥离，与 Apple、VLSI Technology 三方在英国剑桥合资成立 ARM Ltd，原本是为 Apple Newton 掌上电脑设计低功耗芯片。Newton 失败后 ARM 转向"卖芯片图纸"——只收授权费和版税，让各路人马快速生产芯片。这个商业模式在 90 年代半导体行业极其领先，让 ARM 在手机时代形成几乎垄断：今天市面上超过 99% 的手机 CPU 都是 ARM 架构；35 年累计基于 ARM 架构生产的芯片超过 3500 亿枚。2026 年 ARM 破天荒发布自己的第一颗自研芯片 **Aegis CPU**，35 年来首次站到台前。

**RISC-V — 2010 年伯克利学派，做芯片界的 Linux**：2010 年由加州大学伯克利分校（UC Berkeley）的教授们发起，目标很简单甚至有点理想主义——做芯片界的 Linux，做一套任何人都能用、任何人都能扩展、任何人都不用交巨额授权费的开放指令集架构。前十几年 RISC-V 被视为"低端平替、MCU 专业户"，只能在嵌入式/物联网/SSD 主控这些低端领域打杂；2026 年 3 月阿里达摩院玄铁 C950 在选铁生态大会发布——直接把 RISC-V 拎上了高性能服务器主战场。

## § 3 AI/Agent 时代的算力新需求

**写作目标**：解释 AI/Agent 大爆发为什么会**重新洗牌**算力需求——这是后面 § 4 RISC-V 窗口期论证的前提。

**必答的关键问题**：

- 推理（inference）和训练（training）对芯片的需求有什么本质区别？
- Agent 时代为什么 token 消耗量会暴涨？（多轮规划、工具调用、长上下文、自我反思）
- 边缘 Agent（手机/笔记本/IoT 本地跑 LLM）vs 云端 Agent，对芯片的不同要求？
- 内存带宽墙（memory wall）为什么是 LLM 时代的硬约束？
- 专用加速器（NPU / DSA）为什么越来越流行？SoC 设计自由度为什么重要？
- 这一切对"指令集是不是开源"有没有影响？

**建议字数**：700 字

<!-- PLACEHOLDER_START -->

**训练 vs 推理的本质区别——黄仁勋的"教授 vs 小学生"比喻**：CPU 像一个擅长复杂演算+逻辑推理的资深教授；GPU 像一群只会简单算术的小学生。过去 2-3 年训练万亿参数模型本质是"几十亿次简单算术"——一个老教授根本不够，几万个小学生齐刷刷干就赢——这是 Nvidia 神级崛起的本质。但 **2026 年游戏规则变了**。

**Agent 时代 token 消耗为什么暴涨**：Agentic AI 不再是 one-shot 问答（chat box 给你答案就结束），而是给它一个任务让它分解目标、做逻辑推理、调用十几个工具、出错时自我调整、跨多个步骤迭代执行（典型如最近大火的 Manus/openCloud）。在计算机科学语言里，这是"复杂控制流逻辑+条件分支+高频网络 I/O"，是严格**串行的任务**（去图书馆找特定的书→翻到第 10 页→根据线索找钥匙→打开密码锁柜子，上一步没做完下一步根本起不来）。单次任务的 token 消耗比传统 chatbot 高 10 倍到 100 倍——「**没有效率，AGI 就只能是个 demo；有了效率，AGI 才能成为真正的产品和基础设施**」（肖志斌，zFlow AI 创始人，硅谷101 播客 2026.04）。

**CPU/GPU 比值的历史性翻转——600 亿美元增量市场的来源**：过去服务器里 CPU/GPU 比值固定在 1:8（8 张 H100 配 1 颗 Xeon，CPU 只做装载数据的粗活）。但 2026 年陈立武在 Intel 财报会上披露：**Agentic Computing 部署里 CPU:GPU 已紧密到 1:4，正快速逼近 1:1 黄金分割**。摩根士丹利预测此结构性变化将为服务器 CPU 创造 **600 亿美元增量市场**。研究数据显示 **Agent 工作负载中 CPU 处理工具调用的延迟占比高达 90.6%**——CPU 已经成为 AI 智能体时代真正的算力瓶颈，不是 GPU。Nvidia 自己也变了：新一代 AI 服务器从"8 张 GPU + 2 张 x86 CPU"换成"8 张 GPU + 4 张 Grace CPU"，整柜 36 颗自研 ARM CPU，正是因为 GPU 算力越来越强、CPU 调度跟不上就会成为瓶颈，白白浪费算力和功耗。

**内存带宽墙 + TTFT（Time to First Token）成为新指标**：LLM 推理瓶颈在内存带宽不在算力，HBM、LPDDR5X 等带宽派系成为关键。ARM Aegis CPU 把每核 6GB/s 内存带宽、<100ns 缓存延迟做到了顶配 x86 CPU 的 2 倍、TDP 功耗减少 40%。Agent 时代行业不再只看 FLOPS，而看 TTFT——人机交互的终极延迟指标。"高端 CPU + 高带宽内存"在 Agent 复杂逻辑任务上比 GPU 更快、更便宜。

**边缘 Agent 与"应用反向定义架构"**：LLM 训练集中在 H100/B200 少数玩家手里，但 LLM 推理（特别是 Agent 反复推理+工具调用）将走向千千万万的终端——手机、笔记本、汽车、机器人、AR/AI 眼镜。能不能本地跑、推理便宜不便宜，成为第一性问题。专用加速器（NPU/矩阵乘法单元/稀疏化引擎）崛起，CPU 只负责调度。能不能在 CPU 旁边自由集成自定义加速器、定制 cache 一致性、定制中断架构，变成核心竞争力。x86 收授权费、ARM 收 IP 费且即使买 IP 授权也是"买房不是买地"（你可以装修、不能砸承重墙、更别想改地基）；只有 RISC-V 从指令集层面允许"自己改地基"——这就是"应用反向定义架构"的设计自由度，恰恰是开源指令集的天然优势。

**DeepSeek V4 的范式启示——模型架构与硬件协同**：2026 年 4 月 DeepSeek V4 发布，三大核心技术（SCA+HCA 混合注意力降低长上下文推理成本、MHC 多通道残差稳定深层训练、MUON 优化器加速收敛）本质都在"模型架构 ↔ 硬件架构协同设计"上做精细化文章——长上下文场景计算成本降到原来的 1/3、KV cache 降到 1/10。这证明了"**模型不仅是参数量比赛，而是 token efficiency 比赛**"。DeepSeek V4 已被报道适配华为升腾、寒武纪等国产芯片，部分 V4 Flash 训练也使用了华为芯片——非英伟达芯片在推理 workload 上首次有了大规模承接的可能。

<!-- PLACEHOLDER_END -->

**默认占位**（朴素版，仅在留白区为空时启用）：

LLM 训练是少数公司的事，集中在 Nvidia H100/B200 之类的旗舰 GPU 上；但 LLM 推理（特别是 Agent 反复推理 + 工具调用）会**走向千千万万的终端设备**——手机、笔记本、汽车、机器人。Agent 时代的 token 消耗暴涨：一个简单任务可能要多轮规划、调用十几个工具、反复反思修正，单次任务的 token 消耗比传统 chatbot 高 1-2 个数量级。这把"推理便宜不便宜、能不能本地跑"推到第一性问题。芯片需求随之分化：(1) 内存带宽墙——LLM 推理瓶颈在内存带宽不在算力，HBM、LPDDR5X 等带宽派系成为关键；(2) 专用加速器——NPU、矩阵乘法单元、稀疏化引擎崛起，CPU 只负责调度；(3) SoC 设计自由度——能不能在 CPU 旁边自由集成自定义加速器、定制 cache 一致性、定制中断架构，变成核心竞争力。这种"设计自由度"恰恰是开源指令集（RISC-V）的天然优势——x86 / ARM 都收授权费、都有架构约束，RISC-V 你想加什么加什么。

## § 4 RISC-V 在 Agent 时代的窗口期

**写作目标**：这是 demo 的**核心论点段落**——回答用户最关心的"RISC-V 能不能借 Agent 大爆发翻身"。要兼顾正方论据和反方质疑。

**必答的关键问题**：

- 开源带来的设计灵活性具体能省多少钱、省多长时间？
- "不被锁喉"的定制权对中国国产替代叙事意味着什么？甲辰计划具体目标？
- Nvidia GPU 内嵌入式控制核迁向 RISC-V，是不是 RISC-V 大规模落地的暗线？
- 苹果当年自研芯片时为什么选 ARM 不选 RISC-V？（关键问题）如果 2030 年苹果再做一次选择会怎样？

**正方四条窗口期论据**：

**(1) 设计自由度 = 应用反向定义架构**：Agent 推理对内存带宽和专用加速器极度敏感，能自由扩展指令集 = 能定制最适合自己 workload 的芯片。**玄铁 C950 是经典样本**——RVV（RISC-V Vector）向量引擎宽度做到 4096 位、自研矩阵加速引擎单芯片做到 8TOPS 算力、走"**通推一体**"路线（把推理算力直接放进 CPU 侧避免 CPU-GPU 数据来回拷贝）；已跑通千问 3 满血版 34 token/s、DeepSeek V3 满血版 18 token/s。达摩院首席科学家孟建议博士的金句：「**以前是架构定义应用——你得适应 CPU 的规则；现在是应用反向定义架构——尤其在 AI Agent 时代，每个公司都有自己的模型形态、工具链组合、场景约束，别人家的最优解未必是你的最优解。**」这是从"租客"到"房东"的区别——租客可以软装，房东可以改户型。

**(2) 不被锁喉 = 架构主权**：黄仁勋讲"主权 AI"——各国都在建设属于自己的 AI 能力；但孟博进一步把它推到底层：不是"我能买到芯片的主权"，而是"**我能决定芯片底层规则的主权**" = **架构主权**。中国国产替代叙事下，RISC-V 是甲辰计划的国家级押注，50+ 家中国企业加入。为了打通"芯片设计→制造→软件适配"的全链路，达摩院组建了"无锡联盟"，让 RISC-V 笔记本能跑 WPS、钉钉等基础应用。ARM 和高通这几年的纠纷表面看是授权和合同的争议，深层看就是一句话：**在专有架构模式下，即便你有极强的设计能力，你的命脉依然可能握在别人手里**。

**(3) 嵌入式控制核暗线已经成军**：Nvidia GPU 内嵌的 Falcon 控制核已迁向 RISC-V；西部数据等存储厂商把 RISC-V 用作 SSD 主控；汽车/工业/IoT 大量 MCU 用 RISC-V 已是常态。这条"MCU 级别 RISC-V"的累计出货量已达**十亿颗**级别。达摩院 7 年 14 款 CPU、进入 250+ 款量产芯片、总出货 **45 亿颗**——证明 RISC-V 下沉市场已经跑通商业闭环。但与"手机应用处理器/服务器 CPU 级别"要严格区分，不能直接外推到旗舰场景。

- **训练侧**：长期来看 GPU 仍是 default，CUDA 生态短期无法被替代。

## § 5 三个核心质疑

**写作目标**：把 RISC-V 反方质疑摆台面上，给陈立武（x86）/ 库克（ARM）/ 黄仁勋（中立）提供反驳吴伟的弹药。不要回避现实困难。

**必答的关键问题**：

- **生态质疑**：操作系统支持、编译器/工具链成熟度、CUDA 这种行业标准的不可替代性
- **性能质疑**：RISC-V 旗舰核与 ARM Cortex-X / Apple M 系列 / Intel Xeon 的 SPEC 跑分差距实际有多大？什么时候追平？
- **商业模式质疑**：开源指令集靠什么赚钱？SiFive / 阿里玄铁 / 香山的商业闭环都跑通了吗？长期可持续吗？

**建议字数**：500 字

<!-- PLACEHOLDER_START -->

**(1) 生态质疑 —— CUDA + 碎片化双重困境**：
- **CUDA 不可替代**：CUDA 不只是 GPU 指令集，而是 20 年累积的库、工具链、开发者社区——RISC-V 即使硬件层追上也短期无法复制。Nvidia 的真正护城河不是 GPU 而是 CUDA 生态、NVLink、InfiniBand、整套通信库和供应链。
- **OS/编译器层**：Linux 主线对 RISC-V 已成熟，但 Windows 旗舰桌面级支持仍弱；LLVM/GCC 主线都进了 RISC-V，但向量扩展（RVV 1.0）的优化成熟度仍落后 ARM SVE / x86 AVX 多个版本。
- **碎片化问题真实存在**：如果每家都从零造轮子、定制指令集、上层软件互相不兼容、语言不通，生态合力就形不成。达摩院的解法是 **flex 软件平台 + 无锡联盟**：在公认开源核之上做 20% 差异化扩展产生 80% 价值，底层兼容性由玄铁这帮人统一搞定。这条路能不能跑通仍待时间验证。

**(2) 性能质疑 —— "逼近"不等于"追平"**：
- 2026 年 3 月玄铁 C950 已在 SPECint2006 总分突破 70 分（每 GHz 22 分、5nm 工艺、主频 3.2 GHz），**逼近** Intel Xeon 6 / ARM Neoverse V2 / AMD Zen5——但与 ARM Cortex-X4 / Apple M5 / AMD Zen5 旗舰仍有显著差距。
- **x86 阵营的最猛反击**：Intel **Panther Lake** 在 2026 年初的权威测试中超过 Apple M5 的图形性能、续航 14.5-15 小时——彻底打破了"x86 低功耗打不过 ARM"的刻板印象。
- **物理极限层面的差距**：Intel 18A 的 **PowerVia 背面供电**把电压跌落从 7% 降到 1% 以下、整体功耗降 8%，比台积电领先 12-18 个月（台积电 1.6nm 背面供电要等 2026 下半年）；Intel 14A 直接上 ASML 的 4 亿美元 **High-NA EUV TWINSCAN EXE:5200B**（0.55 NA，单次 EUV 曝光 8nm 分辨率，消除多重曝光的良率问题）——这种"硬件物理极限+先进光刻"的能力，RISC-V 阵营在短期内无法复制。
- **追平时间**：RISC-V 旗舰核追平 ARM/x86 旗舰预计 **2028-2030**。

**(3) 商业模式质疑 —— 连 ARM 都觉得授权费不够吃了**：
- **SiFive** 走 IP 授权（类似 ARM Ltd）路径，近年估值上涨但盈利仍存疑。
- **中国阵营** 走"国家级押注+设备厂商免授权费节省"路径，长期闭环依赖国产替代政策强度。
- **Tenstorrent**（Jim Keller 领导，前 AMD 总架构师）走"AI 加速器+RISC-V 主控"组合拳，2025 年底刚发布 Ascalon SCX。
- **致命信号**：35 年来 ARM 第一次"卖图纸卖到自己下场造芯片"——2026 年发布 Aegis CPU 进军 AI 数据中心。这反过来说明：**"开源 IP 只授权"的商业模式在 AI 数据中心时代正面临巨大压力**。如果连 ARM 都觉得授权费不够吃了要亲自下场，RISC-V 阵营靠什么赚到足以维持长期研发投入的钱？这是悬在 RISC-V 商业模式头上的剑。

(1) **生态**——CUDA 不只是 GPU 指令集，是 20 年累计的库、工具链、开发者社区，RISC-V 即使在硬件层追上也短期复制不了；OS 这边 Linux 主线对 RISC-V 已经成熟，但 Windows 旗舰桌面级支持仍弱；编译器 LLVM / GCC 主线都进了，但向量扩展（RVV 1.0）的优化成熟度仍落后 ARM SVE / x86 AVX。(2) **性能**——2026 年 RISC-V 旗舰核（阿里玄铁 C950、香山昆明湖）SPECint2006 突破 70 区间，已经接近 ARM Cortex-A77 / Intel Skylake 区间，但与 ARM Cortex-X4 / Apple M3 / AMD Zen5 旗舰仍有显著差距；追平旗舰预计 2028-2030。(3) **商业模式**——SiFive 走 IP 授权（类似 ARM Ltd），近年估值上涨但盈利仍存疑；中国阵营走"国家级押注 + 设备厂商免授权费节省"路径，长期闭环依赖国产替代政策强度；Tenstorrent（前 AMD 总架构师 Jim Keller 领导）押注 AI 加速器 + RISC-V 主控的组合拳。
